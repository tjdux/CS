## API URI 설계

- 제일 중요한 것은 **리소스 식별**
- URI 계층 구조 활용
- 리소스: 개념적 대상 (e.g. 회원 목록 조회라면 회원이 리소스)
- API URI는 리소스만 식별하면 됨 (e.g. 회원 리소스를 URI에 매핑)
- API URI 설계
  - 회원 목록 조회: `/members`
  - 회원 조회: `/members/{id}`
  - 회원 등록: `/members/{id}`
  - 회원 수정: `/members/{id}`
  - 회원 삭제: `/members/{id}`
- 계층 구조상 상위를 컬렉션으로 보고 복수단어 사용 권장

<br/>

## 리소스와 행위를 분리

- **URI는 리소스만 식별**
- 리소스 (명사)와 해당 리소스를 대상으로 하는 행위(동사)를 분리
  - 리소스: 회원
  - 행위: 조회, 등록, 수정, 삭제
- 행위는 HTTP 메소드로 구분
